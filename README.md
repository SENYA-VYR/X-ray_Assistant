# 🦴 Рентген-ассистент (X-ray Assistant)

**Веб-платформа для автоматического обнаружения трещин и микротрещин на рентгеновских снимках костей с обучаемой обратной связью от врача.**

---

## 📌 О проекте

Веб-приложение помогает врачам-рентгенологам выявлять трещины и микротрещины на рентгеновских снимках. Нейросеть анализирует снимок, выделяет подозрительные зоны и оценивает вероятность перелома. Врач может подтвердить или опровергнуть результат — на основе этих ответов нейросеть **дообучается**, повышая точность с каждым исследованием.

---

## 🩻 Что такое микротрещина?

**Микротрещина** — частичное нарушение целостности костной ткани, при котором образуется линейная или мелкая щель.

### Причины появления:
- Повторяющиеся микротравмы (спорт, физические нагрузки)
- Остеопения и остеопороз
- Анатомические особенности
- Механические травмы

### Методы диагностики:
| Метод | Возможности |
|-------|-------------|
| **Рентген** | Показывает наличие трещины, её тип и местоположение |
| **МРТ** | Детальная визуализация костных структур |
| **КТ** | Более детальное изображение, чем рентген |

## 🎯 Целевая аудитория

### Прямая:
- **Медицинские учреждения** (травматология, ортопедия, рентгенология)
- **Врачи-рентгенологи** — инструмент для перепроверки диагноза

### Косвенная:
- **Спортсмены и люди с высокими физическими нагрузками** — группа риска по стресс-переломам
- **Пожилые люди** — возрастное снижение плотности костной ткани

---

## 🧠 Как работает нейросеть

### Алгоритм анализа снимка:

1. **Предобработка:** снимок переводится в ч/б, увеличивается контраст, сжимается до 256×256 пикселей
2. **Анализ:** нейросеть для каждого пикселя определяет вероятность наличия трещины 
3. **Постобработка:** пиксели с вероятностью выше порога группируются в зоны, отсеивается шум
4. **Визуализация:** зоны выделяются цветными рамками в зависимости от вероятности

### Цветовое кодирование:
- 🔴 **Красный** — высокая вероятность (>65%)
- 🟠 **Оранжевый** — средняя вероятность (35–64%)
- 🟡 **Жёлтый** — низкая вероятность (<35%)

### Обучение на обратной связи:

Врач подтверждает (✅ трещина) или опровергает (❌ ложное срабатывание) каждую зону → нейросеть сравнивает свой ответ с ответом врача → вычисляет ошибку → корректирует веса → сохраняет улучшенную модель → **становится точнее**

РЕНТГЕНОЛОГ
│
├─ Авторизация/Регистрация ──→ Flask (сессия) ──→ SQLite (doctors)
│
├─ Новое исследование:
│ ├─ Выбор пациента ──→ Flask ──→ SQLite (patients)
│ └─ Загрузка снимка ──→ static/uploads/
│
├─ Анализ снимка:
│ └─ Flask ──→ model.py ──→ OpenCV + Нейросеть
│ │
│ └─ Объединение результатов → снимок с рамками + JSON
│ │
│ └─ Flask ──→ result.html (визуализация)
│
├─ Обратная связь:
│ └─ Врач оценивает каждую зону (да/нет)
│ │
│ ├─ Flask ──→ SQLite (training_data)
│ └─ model.py → дообучение нейросети → crack_model.pth
│
└─ Просмотр истории:
└─ Flask ──→ SQLite (studies) ──→ dashboard.html

## 📋 База данных

### Таблицы:

| Таблица | Назначение |
|---------|------------|
| **doctors** | Врачи: логин, пароль (хэш), ФИО, медучреждение |
| **patients** | Пациенты: ФИО, дата рождения |
| **studies** | Исследования: ссылка на врача и пациента, дата, пути к снимкам, результаты |
| **training_data** | Данные для обучения: координаты зон, вердикт врача |

### Фрагмент модели Study:

```python
class Study(db.Model):
    __tablename__ = 'studies'
    id = db.Column(db.Integer, primary_key=True)
    doctor_id = db.Column(db.Integer, db.ForeignKey('doctors.id'), nullable=False)
